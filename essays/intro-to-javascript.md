## Thoughts on Javascript

I have not done any proper programming for a little over a year and the last time I have was in C. So, when I first started learning Javascript, it was definitely a little rough to say the least. However, I have light experience in Java which helped me to understand as well as the act of coding itself started to refresh my memory on syntax and basic coding rules. As I started to learn more about Javascript, I started to dislike previous programming languages I have learned. In my opinion, I find Javascript easy and enjoyable to use, which is something that can be said for a lot of programming languages because of all the rules and regulations one must follow. However, with Javascript, I feel like I have the freedom to almost anything. Whether this is beneficial or detrimental to me in software engineering is yet to be determined but I am satisfied with my experience so far.

## New Things I've Learned

Since I have prior experience in programming in other languages, I mostly learned new things about Javascript itself rather than the basics of syntax, loops, and data types among other things. Firstly, I learned that functions are first class, which was something very different and new to me. In short, when a function is first-class, they can be treated like any other variable, they can be passed as an argument to other functions, and they can be returned by other functions. This is extremely useful because it allows for so much more flexibility when it comes to manipulating functions. Addtionally, I learned that inheritance is simpler in Javascript. Interfaces, abstract classes, final classes, virtual base classes, deconstructors, et cetera, are no longer needed in Javascript. It is also more flexible and expressive which means that as data is encountered, the structure of classes can change and evolve to manipulate that data. While I am still wrapping my head around the last point, it sounds like very useful yet different from how I have learned about classes and inheritance in my previous courses.

## Athletic Software Engineering

Athletic Software Engineering is a learning pedagogy developed my Professor Philip Johnson to help students learn software engineering in a similar way to how athletes train in order for them to become used to high intensity, time-constrained, and stress-inducing problems. Similar to how an athlete trains a skill, one must keep repeating an exercise until it becomes natural or muscle-memory. In the case of software engineering, this is presented in the form of WODs and practice WODs, that are designed to test our skills and knowledge in a closed, time-constrained, environment. While it is definitely stressful in the moment, I find them very useful and totally atypical from how I have previously studied computer science. As long as one puts in the time and effort, a skill can be easily developed or possibly mastered in a timely manner. In addition, I feel that I will get used to the stress and time pressure, which will definitely prepare me for future situations where those skills are necessary. I could even apply this method of learning to other courses or other aspects of my life.
